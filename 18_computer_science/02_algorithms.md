# Алгоритмы — обзор для QA

## Обзор

Алгоритмы — это пошаговые инструкции для решения задач. Для QA Automation Engineer глубокое знание алгоритмов требуется реже, чем для backend-разработчика, однако базовое понимание необходимо: алгоритмические задачи регулярно встречаются на технических интервью; понимание сложности алгоритмов помогает писать эффективный тестовый код; при тестировании production-кода важно понимать, как работают основные алгоритмы, чтобы проектировать граничные тест-кейсы. В этом разделе рассматриваются алгоритмы сортировки, поиска и основы алгоритмического мышления.

## Алгоритмы сортировки

Сортировка — одна из самых фундаментальных задач в программировании. Знание разных алгоритмов сортировки важно для понимания сложности (`Big O`) и для ответов на интервью.

### Bubble Sort (пузырьковая сортировка)

**Принцип:** многократно проходим по массиву, сравнивая соседние элементы и меняя их местами, если они стоят в неправильном порядке. «Большие» элементы «всплывают» наверх, как пузырьки.

```java
/**
 * Пузырьковая сортировка.
 * Сложность: O(n²) — в среднем и худшем случае.
 * O(n) — в лучшем случае (массив уже отсортирован, с оптимизацией).
 */
public static void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        boolean swapped = false; // Флаг оптимизации
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                // Меняем соседние элементы местами
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = true;
            }
        }
        // Если за проход ни одной перестановки — массив отсортирован
        if (!swapped) break;
    }
}
```

**Пример пошагово** (массив `[5, 3, 8, 1]`):
```
Проход 1: [3, 5, 1, 8]  — 8 «всплыл» на конец
Проход 2: [3, 1, 5, 8]  — 5 встал на место
Проход 3: [1, 3, 5, 8]  — отсортировано
```

### Selection Sort (сортировка выбором)

**Принцип:** на каждом шаге находим минимальный элемент в неотсортированной части массива и ставим его на своё место.

```java
/**
 * Сортировка выбором.
 * Сложность: O(n²) — всегда (даже для отсортированного массива).
 */
public static void selectionSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        int minIndex = i; // Предполагаем, что текущий элемент — минимальный
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIndex]) {
                minIndex = j; // Нашли элемент меньше
            }
        }
        // Меняем местами текущий элемент и найденный минимум
        int temp = arr[i];
        arr[i] = arr[minIndex];
        arr[minIndex] = temp;
    }
}
```

**Пример пошагово** (массив `[5, 3, 8, 1]`):
```
Шаг 1: минимум = 1 (индекс 3) → [1, 3, 8, 5]
Шаг 2: минимум = 3 (индекс 1) → [1, 3, 8, 5]  (уже на месте)
Шаг 3: минимум = 5 (индекс 3) → [1, 3, 5, 8]
```

### Merge Sort (сортировка слиянием)

**Принцип:** алгоритм «разделяй и властвуй» (divide and conquer). Массив рекурсивно делится пополам, каждая половина сортируется, затем отсортированные половины сливаются в одну.

```java
/**
 * Сортировка слиянием.
 * Сложность: O(n log n) — всегда.
 * Дополнительная память: O(n).
 */
public static void mergeSort(int[] arr, int left, int right) {
    if (left < right) {
        int mid = (left + right) / 2;

        // Рекурсивно сортируем левую и правую половины
        mergeSort(arr, left, mid);
        mergeSort(arr, mid + 1, right);

        // Сливаем отсортированные половины
        merge(arr, left, mid, right);
    }
}

private static void merge(int[] arr, int left, int mid, int right) {
    // Создаём временные массивы для левой и правой частей
    int[] leftArr = Arrays.copyOfRange(arr, left, mid + 1);
    int[] rightArr = Arrays.copyOfRange(arr, mid + 1, right + 1);

    int i = 0, j = 0, k = left;

    // Сливаем, выбирая меньший элемент из двух частей
    while (i < leftArr.length && j < rightArr.length) {
        if (leftArr[i] <= rightArr[j]) {
            arr[k++] = leftArr[i++];
        } else {
            arr[k++] = rightArr[j++];
        }
    }

    // Копируем оставшиеся элементы
    while (i < leftArr.length) arr[k++] = leftArr[i++];
    while (j < rightArr.length) arr[k++] = rightArr[j++];
}
```

**Визуализация:**
```
[5, 3, 8, 1]
     ↓ разделить
[5, 3]   [8, 1]
   ↓         ↓ разделить
[5] [3]   [8] [1]
   ↓         ↓ слить
[3, 5]   [1, 8]
     ↓ слить
[1, 3, 5, 8]
```

### Quick Sort (быстрая сортировка)

**Принцип:** выбирается опорный элемент (pivot), массив разделяется на две части — элементы меньше pivot и больше pivot. Каждая часть сортируется рекурсивно.

```java
/**
 * Быстрая сортировка.
 * Сложность: O(n log n) — в среднем; O(n²) — в худшем случае.
 * Дополнительная память: O(log n) — для стека рекурсии.
 */
public static void quickSort(int[] arr, int low, int high) {
    if (low < high) {
        // Разделяем массив относительно опорного элемента
        int pivotIndex = partition(arr, low, high);

        // Рекурсивно сортируем левую и правую части
        quickSort(arr, low, pivotIndex - 1);
        quickSort(arr, pivotIndex + 1, high);
    }
}

private static int partition(int[] arr, int low, int high) {
    int pivot = arr[high]; // Опорный элемент — последний
    int i = low - 1;       // Индекс для элементов меньше pivot

    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            // Меняем местами arr[i] и arr[j]
            int temp = arr[i];
            arr[i] = arr[j];
            arr[j] = temp;
        }
    }

    // Ставим pivot на правильное место
    int temp = arr[i + 1];
    arr[i + 1] = arr[high];
    arr[high] = temp;

    return i + 1;
}
```

### Сравнительная таблица алгоритмов сортировки

| Алгоритм | Лучший случай | Средний | Худший | Память | Стабильная? |
|----------|--------------|---------|--------|--------|-------------|
| Bubble Sort | `O(n)` | `O(n²)` | `O(n²)` | `O(1)` | Да |
| Selection Sort | `O(n²)` | `O(n²)` | `O(n²)` | `O(1)` | Нет |
| Merge Sort | `O(n log n)` | `O(n log n)` | `O(n log n)` | `O(n)` | Да |
| Quick Sort | `O(n log n)` | `O(n log n)` | `O(n²)` | `O(log n)` | Нет |

**Стабильная сортировка** — сохраняет относительный порядок элементов с одинаковыми ключами. Важно, например, при сортировке результатов тестов: если два теста имеют одинаковый статус, стабильная сортировка сохранит их исходный порядок.

**Что использует Java:**
- `Arrays.sort()` для примитивов — Dual-Pivot Quick Sort (`O(n log n)`)
- `Arrays.sort()` для объектов — TimSort (гибрид Merge Sort + Insertion Sort, `O(n log n)`)
- `Collections.sort()` — TimSort

## Алгоритмы поиска

### Linear Search (линейный поиск)

**Принцип:** последовательный перебор всех элементов от начала до конца.

```java
/**
 * Линейный поиск.
 * Сложность: O(n).
 * Работает на любом массиве (отсортированном и нет).
 */
public static int linearSearch(int[] arr, int target) {
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == target) {
            return i; // Нашли — возвращаем индекс
        }
    }
    return -1; // Не нашли
}
```

**Когда использовать:** для небольших коллекций или когда массив не отсортирован и сортировка нецелесообразна.

### Binary Search (бинарный поиск)

**Принцип:** работает только на **отсортированном** массиве. На каждом шаге сравниваем искомый элемент со средним — если меньше, ищем в левой половине; если больше — в правой. Каждый шаг отсекает половину элементов.

```java
/**
 * Бинарный поиск.
 * Сложность: O(log n).
 * ВАЖНО: массив должен быть отсортирован!
 */
public static int binarySearch(int[] sortedArr, int target) {
    int left = 0;
    int right = sortedArr.length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2; // Безопасное вычисление середины

        if (sortedArr[mid] == target) {
            return mid; // Нашли
        } else if (sortedArr[mid] < target) {
            left = mid + 1; // Ищем в правой половине
        } else {
            right = mid - 1; // Ищем в левой половине
        }
    }

    return -1; // Не нашли
}
```

**Пример пошагово** (массив `[1, 3, 5, 8, 12, 15, 20]`, ищем `12`):
```
Шаг 1: mid = 3, arr[3] = 8 < 12 → ищем в правой части [12, 15, 20]
Шаг 2: mid = 5, arr[5] = 15 > 12 → ищем в левой части [12]
Шаг 3: mid = 4, arr[4] = 12 = 12 → нашли! Индекс 4
```

**Важно:** формула `mid = left + (right - left) / 2` безопаснее, чем `(left + right) / 2`, потому что предотвращает целочисленное переполнение при больших значениях `left` и `right`.

### Linear Search vs Binary Search

| Критерий | Linear Search | Binary Search |
|----------|--------------|---------------|
| Сложность | `O(n)` | `O(log n)` |
| Требование | Нет | Массив отсортирован |
| Для 1 000 000 элементов | До 1 000 000 сравнений | До ~20 сравнений |
| Реализация | Проще | Сложнее (легко ошибиться) |
| Готовое решение в Java | — | `Arrays.binarySearch()`, `Collections.binarySearch()` |

## Основы алгоритмического мышления для QA

### Принцип «Разделяй и властвуй» (Divide and Conquer)

Разбить большую задачу на меньшие подзадачи, решить каждую отдельно, объединить результаты.

**Применение в QA:**
- Разбиение большого тестового набора на группы (smoke, regression, integration)
- Декомпозиция сложного тест-кейса на отдельные шаги
- Поиск причины бага методом бинарного поиска (git bisect)

### Жадные алгоритмы (Greedy)

На каждом шаге выбирать локально оптимальное решение.

**Применение в QA:**
- Приоритизация тестов: сначала запускать тесты, которые чаще выявляют баги
- Выбор минимального набора тест-кейсов для покрытия максимума требований

### Рекурсия

Функция вызывает сама себя для решения подзадачи.

```java
/**
 * Пример рекурсии: обход дерева элементов на веб-странице.
 * Полезно для QA при проверке вложенных структур (меню, деревья, комментарии).
 */
public static void findAllElements(WebElement parent, List<WebElement> result) {
    List<WebElement> children = parent.findElements(By.xpath("./*"));
    for (WebElement child : children) {
        result.add(child);
        // Рекурсивный вызов для каждого потомка
        findAllElements(child, result);
    }
}
```

### Два указателя (Two Pointers)

Техника, при которой два указателя движутся по массиву навстречу друг другу или в одном направлении.

```java
/**
 * Проверка, является ли строка палиндромом.
 * Полезно для QA: валидация данных, тестирование строковых алгоритмов.
 */
public static boolean isPalindrome(String s) {
    int left = 0;
    int right = s.length() - 1;

    while (left < right) {
        if (s.charAt(left) != s.charAt(right)) {
            return false; // Символы не совпадают — не палиндром
        }
        left++;
        right--;
    }
    return true; // Все символы совпали — палиндром
}
```

## Где алгоритмы появляются на интервью QA

### Типичные задачи на собеседованиях

1. **Развернуть строку / массив** — проверка базового понимания циклов и массивов
2. **Найти дубликаты в массиве** — использование HashSet (O(n)) vs вложенный цикл (O(n²))
3. **Проверить палиндром** — работа со строками, два указателя
4. **Найти два числа с заданной суммой** — HashMap для O(n) решения
5. **FizzBuzz** — базовая логика, условия, циклы
6. **Найти максимум / минимум** — перебор с хранением текущего результата

### Пример: найти два числа с заданной суммой

```java
/**
 * Найти пару элементов в массиве, дающих в сумме target.
 * Эффективное решение через HashMap — O(n).
 */
public static int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> seen = new HashMap<>(); // значение → индекс

    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i]; // Какое число нужно для пары
        if (seen.containsKey(complement)) {
            return new int[]{seen.get(complement), i}; // Нашли пару
        }
        seen.put(nums[i], i); // Запоминаем текущий элемент
    }

    return new int[]{}; // Пара не найдена
}
```

### Пример: FizzBuzz

```java
/**
 * Классическая задача FizzBuzz.
 * Часто встречается на собеседованиях как проверка базовой логики.
 */
public static List<String> fizzBuzz(int n) {
    List<String> result = new ArrayList<>();
    for (int i = 1; i <= n; i++) {
        if (i % 15 == 0) {
            result.add("FizzBuzz"); // Делится и на 3, и на 5
        } else if (i % 3 == 0) {
            result.add("Fizz");     // Делится на 3
        } else if (i % 5 == 0) {
            result.add("Buzz");     // Делится на 5
        } else {
            result.add(String.valueOf(i));
        }
    }
    return result;
}
```

## Типичные ошибки

1. **Бинарный поиск на неотсортированном массиве** — результат непредсказуем. Binary search требует предварительной сортировки.

2. **Переполнение при вычислении mid** — `(left + right) / 2` может переполнить `int`. Безопасная формула: `left + (right - left) / 2`.

3. **Ошибка на единицу (off-by-one)** — типичная ошибка в границах циклов. Особенно часто при реализации binary search (`left <= right` vs `left < right`).

4. **Бесконечная рекурсия** — забытый базовый случай приводит к `StackOverflowError`.

5. **Выбор O(n²) алгоритма при наличии O(n) решения** — например, поиск дубликатов вложенным циклом вместо HashSet.

6. **Игнорирование стабильности сортировки** — при сортировке объектов с несколькими полями нестабильный алгоритм может нарушить порядок.

7. **Не учитывать дополнительную память** — merge sort использует `O(n)` памяти, что может быть критично для больших данных.
