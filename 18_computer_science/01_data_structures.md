# Структуры данных — обзор для QA

## Обзор

Структуры данных — это способы организации и хранения данных в программе. Для QA Automation Engineer понимание структур данных важно по нескольким причинам: правильный выбор коллекций в тестовом коде напрямую влияет на его читаемость и производительность; вопросы по структурам данных — стандартная часть технических интервью; при тестировании production-кода нужно понимать, как устроены данные «под капотом», чтобы проектировать эффективные тест-кейсы. В Java структуры данных представлены через Collections Framework.

---

## Big O Notation (нотация «О большое»)

Прежде чем разбирать конкретные структуры, необходимо понять, как оценивается их эффективность. Big O показывает, как растёт время выполнения операции при увеличении количества элементов.

| Обозначение | Название | Пример | Скорость |
|-------------|---------|--------|----------|
| `O(1)` | Константная | Доступ к элементу HashMap по ключу | Мгновенно |
| `O(log n)` | Логарифмическая | Поиск в TreeMap / binary search | Очень быстро |
| `O(n)` | Линейная | Поиск в неотсортированном списке | Приемлемо |
| `O(n log n)` | Линейно-логарифмическая | Сортировка (merge sort, quick sort) | Нормально |
| `O(n²)` | Квадратичная | Вложенные циклы, bubble sort | Медленно |
| `O(2ⁿ)` | Экспоненциальная | Перебор всех подмножеств | Очень медленно |

### Наглядный пример: сколько операций при n = 1 000 000

| Big O | Операций |
|-------|---------|
| `O(1)` | 1 |
| `O(log n)` | ~20 |
| `O(n)` | 1 000 000 |
| `O(n log n)` | ~20 000 000 |
| `O(n²)` | 1 000 000 000 000 |

**Для QA:** если тест перебирает коллекцию из 10 000 элементов во вложенном цикле — это `O(n²)` = 100 000 000 операций. Тест будет медленным. Правильный выбор структуры данных может ускорить его в тысячи раз.

## Array (массив)

Массив — простейшая структура данных с фиксированным размером и доступом по индексу.

```java
// Создание массива фиксированного размера
String[] browsers = new String[3];
browsers[0] = "Chrome";
browsers[1] = "Firefox";
browsers[2] = "Safari";

// Инициализация при создании
int[] statusCodes = {200, 301, 404, 500};

// Доступ по индексу — O(1)
String firstBrowser = browsers[0]; // "Chrome"

// Длина массива
int length = statusCodes.length; // 4
```

### Сложность операций для Array

| Операция | Сложность | Пояснение |
|----------|----------|-----------|
| Доступ по индексу | `O(1)` | Прямой доступ по адресу в памяти |
| Поиск элемента | `O(n)` | Нужно перебрать все элементы |
| Вставка в конец | `O(1)` | Если есть место |
| Вставка в середину | `O(n)` | Нужно сдвинуть элементы |
| Удаление | `O(n)` | Нужно сдвинуть элементы |

**Для QA:** массивы используются для хранения фиксированных тестовых данных (список браузеров, HTTP-статусов). Для динамических коллекций лучше использовать `ArrayList`.

## ArrayList

`ArrayList` — динамический массив, который автоматически увеличивает размер при добавлении элементов.

```java
import java.util.ArrayList;
import java.util.List;

// Создание списка тестовых URL
List<String> testUrls = new ArrayList<>();
testUrls.add("https://example.com/login");
testUrls.add("https://example.com/register");
testUrls.add("https://example.com/profile");

// Доступ по индексу — O(1)
String firstUrl = testUrls.get(0);

// Поиск элемента — O(n)
boolean hasLogin = testUrls.contains("https://example.com/login"); // true

// Размер списка
int size = testUrls.size(); // 3

// Удаление по индексу — O(n) (сдвиг элементов)
testUrls.remove(0);
```

### Сложность операций для ArrayList

| Операция | Сложность |
|----------|----------|
| `get(index)` | `O(1)` |
| `add(element)` (в конец) | `O(1)` амортизированная |
| `add(index, element)` (в середину) | `O(n)` |
| `remove(index)` | `O(n)` |
| `contains(element)` | `O(n)` |
| `size()` | `O(1)` |

## LinkedList

`LinkedList` — двусвязный список, каждый элемент хранит ссылки на предыдущий и следующий.

```java
import java.util.LinkedList;
import java.util.List;

// Создание LinkedList
List<String> testSteps = new LinkedList<>();
testSteps.add("Открыть страницу логина");
testSteps.add("Ввести логин");
testSteps.add("Ввести пароль");
testSteps.add("Нажать кнопку 'Войти'");

// Добавление в начало — O(1) (преимущество перед ArrayList)
((LinkedList<String>) testSteps).addFirst("Открыть браузер");

// Удаление первого элемента — O(1)
((LinkedList<String>) testSteps).removeFirst();
```

### ArrayList vs LinkedList

| Операция | ArrayList | LinkedList |
|----------|----------|-----------|
| Доступ по индексу | `O(1)` | `O(n)` |
| Добавление в конец | `O(1)` | `O(1)` |
| Добавление в начало | `O(n)` | `O(1)` |
| Вставка в середину | `O(n)` | `O(1)` (если есть итератор) |
| Удаление из середины | `O(n)` | `O(1)` (если есть итератор) |
| Потребление памяти | Меньше | Больше (хранит ссылки) |

**Для QA:** в подавляющем большинстве случаев `ArrayList` — лучший выбор. `LinkedList` оправдан, только если вам нужны частые вставки/удаления в начало списка.

## Stack (стек)

Stack — структура данных по принципу LIFO (Last In, First Out): последний добавленный элемент извлекается первым.

```java
import java.util.ArrayDeque;
import java.util.Deque;

// Стек для хранения истории действий в тесте
// (Рекомендуется ArrayDeque вместо устаревшего Stack)
Deque<String> actionHistory = new ArrayDeque<>();

// push — добавить на вершину стека — O(1)
actionHistory.push("Открыть главную");
actionHistory.push("Перейти в каталог");
actionHistory.push("Открыть карточку товара");

// peek — посмотреть вершину без удаления — O(1)
String current = actionHistory.peek(); // "Открыть карточку товара"

// pop — извлечь с вершины — O(1)
String lastAction = actionHistory.pop(); // "Открыть карточку товара"
```

**Для QA:** стек полезен для реализации паттерна «отмена действий», хранения навигационной цепочки (breadcrumbs) в тестах, отслеживания вложенных вызовов.

## Queue (очередь)

Queue — структура данных по принципу FIFO (First In, First Out): первый добавленный элемент извлекается первым.

```java
import java.util.LinkedList;
import java.util.Queue;

// Очередь тестов на выполнение
Queue<String> testQueue = new LinkedList<>();

// offer — добавить в конец очереди — O(1)
testQueue.offer("LoginTest");
testQueue.offer("RegistrationTest");
testQueue.offer("PaymentTest");

// peek — посмотреть первый элемент без удаления — O(1)
String nextTest = testQueue.peek(); // "LoginTest"

// poll — извлечь первый элемент — O(1)
String currentTest = testQueue.poll(); // "LoginTest"
// Теперь в очереди: ["RegistrationTest", "PaymentTest"]
```

**Для QA:** очередь используется для моделирования очереди задач, порядка выполнения тестов, обработки событий.

## HashMap

`HashMap` — структура данных для хранения пар «ключ — значение» с быстрым доступом по ключу.

### Как работает HashMap (упрощённо)

1. При добавлении пары `put(key, value)` вычисляется hash-код ключа
2. По hash-коду определяется индекс «корзины» (bucket) во внутреннем массиве
3. Если в корзине уже есть элемент с другим ключом (коллизия), элементы хранятся в связном списке (или дереве при большом количестве коллизий)
4. При `get(key)` вычисляется hash-код, находится корзина, извлекается значение

```java
import java.util.HashMap;
import java.util.Map;

// Хранение тестовых данных пользователей
Map<String, String> testUser = new HashMap<>();
testUser.put("username", "testuser@example.com");
testUser.put("password", "P@ssw0rd123");
testUser.put("role", "admin");

// Получение значения по ключу — O(1) в среднем
String username = testUser.get("username"); // "testuser@example.com"

// Проверка наличия ключа — O(1)
boolean hasRole = testUser.containsKey("role"); // true

// Проверка наличия значения — O(n)
boolean hasAdmin = testUser.containsValue("admin"); // true

// Перебор всех пар
for (Map.Entry<String, String> entry : testUser.entrySet()) {
    System.out.println(entry.getKey() + " = " + entry.getValue());
}
```

### Коллизии в HashMap

Коллизия возникает, когда два разных ключа дают одинаковый hash-код (или одинаковый индекс корзины).

```java
// Пример: объекты с одинаковым hashCode(), но разными equals()
// HashMap обрабатывает такие случаи через цепочки (linked list / red-black tree)

// Начиная с Java 8: если в одной корзине > 8 элементов,
// связный список превращается в красно-чёрное дерево (O(log n) вместо O(n))
```

### Сложность операций для HashMap

| Операция | Средний случай | Худший случай |
|----------|---------------|---------------|
| `put(key, value)` | `O(1)` | `O(n)` (все ключи в одной корзине) |
| `get(key)` | `O(1)` | `O(n)` |
| `containsKey(key)` | `O(1)` | `O(n)` |
| `containsValue(value)` | `O(n)` | `O(n)` |
| `remove(key)` | `O(1)` | `O(n)` |

## TreeMap

`TreeMap` — отсортированная Map, основанная на красно-чёрном дереве. Ключи всегда хранятся в отсортированном порядке.

```java
import java.util.Map;
import java.util.TreeMap;

// Хранение результатов тестов, отсортированных по имени
Map<String, String> testResults = new TreeMap<>();
testResults.put("CartTest", "PASSED");
testResults.put("AuthTest", "FAILED");
testResults.put("PaymentTest", "PASSED");
testResults.put("BookingTest", "SKIPPED");

// Элементы автоматически отсортированы по ключу
for (Map.Entry<String, String> entry : testResults.entrySet()) {
    System.out.println(entry.getKey() + ": " + entry.getValue());
}
// Вывод:
// AuthTest: FAILED
// BookingTest: SKIPPED
// CartTest: PASSED
// PaymentTest: PASSED
```

### HashMap vs TreeMap

| Критерий | HashMap | TreeMap |
|----------|---------|--------|
| Порядок элементов | Не гарантируется | Отсортированный по ключу |
| Сложность `get/put` | `O(1)` (среднее) | `O(log n)` |
| `null` ключи | Допускает один `null` | Не допускает `null` |
| Внутренняя структура | Hash-таблица | Красно-чёрное дерево |
| Когда использовать | Быстрый поиск по ключу | Нужен порядок / диапазонные запросы |

## HashSet

`HashSet` — множество уникальных элементов, основанное на `HashMap` (элемент хранится как ключ, а значение — заглушка).

```java
import java.util.HashSet;
import java.util.Set;

// Множество уникальных статус-кодов, встреченных при тестировании API
Set<Integer> statusCodes = new HashSet<>();
statusCodes.add(200);
statusCodes.add(404);
statusCodes.add(200); // Дубликат — не добавится
statusCodes.add(500);

System.out.println(statusCodes.size()); // 3 (200 не дублируется)

// Проверка наличия элемента — O(1)
boolean has404 = statusCodes.contains(404); // true

// Типичное применение: проверка уникальности данных в тестах
List<String> userEmails = getUserEmailsFromDB();
Set<String> uniqueEmails = new HashSet<>(userEmails);

if (userEmails.size() != uniqueEmails.size()) {
    // В базе есть дубликаты email — баг!
    fail("Обнаружены дублирующиеся email в базе данных");
}
```

## TreeSet

`TreeSet` — отсортированное множество уникальных элементов, основанное на `TreeMap`.

```java
import java.util.Set;
import java.util.TreeSet;

// Отсортированный набор времён ответа API (для анализа перформанса)
Set<Long> responseTimes = new TreeSet<>();
responseTimes.add(250L);
responseTimes.add(120L);
responseTimes.add(480L);
responseTimes.add(95L);
responseTimes.add(120L); // Дубликат не добавится

// Элементы отсортированы: [95, 120, 250, 480]
System.out.println(responseTimes);

// Минимальное и максимальное время ответа
System.out.println(((TreeSet<Long>) responseTimes).first()); // 95
System.out.println(((TreeSet<Long>) responseTimes).last());  // 480
```

### HashSet vs TreeSet

| Критерий | HashSet | TreeSet |
|----------|---------|--------|
| Порядок элементов | Не гарантируется | Отсортированный |
| Сложность `add/contains` | `O(1)` (среднее) | `O(log n)` |
| `null` элементы | Допускает один `null` | Не допускает `null` |
| Когда использовать | Быстрая проверка уникальности | Нужен порядок элементов |

## Сводная таблица структур данных

| Структура | Доступ | Поиск | Вставка | Удаление | Порядок | Дубликаты |
|-----------|--------|-------|---------|----------|---------|-----------|
| Array / ArrayList | `O(1)` | `O(n)` | `O(1)`* | `O(n)` | По индексу | Да |
| LinkedList | `O(n)` | `O(n)` | `O(1)` | `O(1)` | По индексу | Да |
| HashMap | `O(1)` | `O(1)` по ключу | `O(1)` | `O(1)` | Нет | Ключи — нет |
| TreeMap | `O(log n)` | `O(log n)` | `O(log n)` | `O(log n)` | Отсортированный | Ключи — нет |
| HashSet | — | `O(1)` | `O(1)` | `O(1)` | Нет | Нет |
| TreeSet | — | `O(log n)` | `O(log n)` | `O(log n)` | Отсортированный | Нет |
| Stack | `O(n)` | `O(n)` | `O(1)` (push) | `O(1)` (pop) | LIFO | Да |
| Queue | `O(n)` | `O(n)` | `O(1)` (offer) | `O(1)` (poll) | FIFO | Да |

\* — добавление в конец ArrayList; в середину — `O(n)`.

## Связь с тестированием

| Сценарий в тестировании | Подходящая структура | Почему |
|------------------------|---------------------|--------|
| Хранение тестовых данных (ключ-значение) | `HashMap` | Быстрый доступ по ключу, O(1) |
| Список шагов теста | `ArrayList` | Порядок важен, доступ по индексу |
| Проверка уникальности данных в БД | `HashSet` | Быстрая проверка дубликатов |
| Отсортированные результаты | `TreeMap` / `TreeSet` | Автоматическая сортировка |
| Навигационная история (back/forward) | `Stack` (Deque) | LIFO — последнее действие первым |
| Очередь запросов при нагрузочном тестировании | `Queue` | FIFO — порядок обработки |
| Параметризация тестов | `List<Object[]>` / `ArrayList` | Упорядоченный набор данных |

## Типичные ошибки

1. **Использование `List` вместо `Set` для проверки уникальности** — `list.contains()` работает за `O(n)`, а `set.contains()` — за `O(1)`. На больших объёмах данных разница огромна.

2. **Не переопределены `equals()` и `hashCode()`** — при использовании собственных объектов как ключей HashMap или элементов HashSet. Без корректных `equals()` и `hashCode()` объекты не будут находиться.

3. **Модификация коллекции во время итерации** — приводит к `ConcurrentModificationException`. Используйте `Iterator.remove()` или `removeIf()`.

4. **Использование `LinkedList` «по привычке»** — в 95% случаев `ArrayList` быстрее и эффективнее по памяти.

5. **Путаница между `null` и пустой коллекцией** — метод возвращает `null` вместо пустого списка. Всегда возвращайте `Collections.emptyList()` вместо `null`.

6. **Неправильная оценка Big O** — утверждение, что `HashMap.get()` всегда `O(1)`. В худшем случае (все элементы в одной корзине) — `O(n)`.

7. **Выбор `TreeMap` без необходимости сортировки** — `TreeMap` медленнее `HashMap` (`O(log n)` vs `O(1)`). Используйте `TreeMap`, только если порядок ключей важен.
