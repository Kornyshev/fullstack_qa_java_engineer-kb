# Полезные ресурсы для QA-инженера

## Обзор

Этот раздел -- структурированный справочник ресурсов для самостоятельного обучения и развития
в QA. Ресурсы отсортированы по категориям и снабжены пояснениями: для кого подходит,
что именно даёт, на каком языке. Приоритет отдан бесплатным и доступным материалам.

Рекомендуемый порядок использования:
1. **Книги** -- для формирования фундамента и понятийного аппарата
2. **YouTube** -- для визуального закрепления и практических демонстраций
3. **Курсы** -- для системного прохождения тем с практикой
4. **Платформы для практики** -- для отработки конкретных навыков
5. **Официальная документация** -- как постоянный справочник при работе
6. **Сообщества** -- для общения, вопросов и отслеживания трендов

## Книги

### Тестирование и QA

#### "Тестирование dot com" -- Роман Савин

- **Язык:** русский
- **Уровень:** начинающий
- **Что даёт:** основы тестирования в формате разговорного языка. Понятия дефектов,
  тест-кейсов, уровней тестирования, процессов в команде. Книга написана живым языком
  с реальными примерами из IT-компаний.
- **Кому читать:** обязательна для всех, кто начинает путь в QA. Даже если вы уже
  знаете теорию -- освежите терминологию перед интервью.
- **Совет:** читайте с карандашом, конспектируйте определения. Многие интервьюеры
  задают вопросы именно в формулировках из этой книги.

#### "Rapid Software Testing" -- James Bach, Michael Bolton

- **Язык:** английский
- **Уровень:** средний / продвинутый
- **Что даёт:** методологию exploratory testing, критическое мышление в тестировании,
  понятие "тестирование как исследование". Авторы -- основоположники context-driven
  подхода в тестировании.
- **Кому читать:** тем, кто хочет выйти за рамки формального "выполнения тест-кейсов"
  и развить навык критического анализа продукта.
- **Совет:** начните с бесплатных статей James Bach на его блоге, затем переходите к книге.

#### "Lessons Learned in Software Testing" -- Cem Kaner, James Bach, Bret Pettichord

- **Язык:** английский
- **Уровень:** средний
- **Что даёт:** 293 урока из реальной практики тестирования. Каждый урок -- 1-2 страницы.
  Охватывает всё: от багов до политики в команде, от автоматизации до управления проектами.
- **Кому читать:** как дополнение к основной книге по тестированию. Отлично подходит
  для чтения по одному уроку в день.

#### "Как тестируют в Google" -- James Whittaker, Jason Arbon, Jeff Carollo

- **Язык:** русский перевод доступен
- **Уровень:** средний
- **Что даёт:** взгляд изнутри на процессы тестирования в Google. Роли SET
  (Software Engineer in Test), TE (Test Engineer), TEM (Test Engineering Manager).
  Подходы к тестированию масштабных систем.
- **Кому читать:** для понимания, как устроено тестирование на уровне enterprise.

### Java и программирование

#### "Java. Библиотека профессионала" -- Cay Horstmann

- **Язык:** русский перевод (Хорстманн)
- **Уровень:** начинающий -> продвинутый (два тома)
- **Что даёт:** полное и фундаментальное изложение Java. Том 1 -- основы: типы данных,
  ООП, Collections, Stream API, исключения, generics. Том 2 -- продвинутые темы:
  многопоточность, JDBC, networking.
- **Кому читать:** обязательна для QA Automation. Не нужно читать от корки до корки --
  сфокусируйтесь на главах, релевантных для автоматизации: ООП, коллекции, Stream API,
  исключения, файловый ввод-вывод.
- **Совет:** читайте главы параллельно с практикой. Прочитали про Stream API --
  сразу напишите 10 упражнений.

#### "Effective Java" -- Joshua Bloch

- **Язык:** английский (есть неофициальные переводы)
- **Уровень:** средний / продвинутый
- **Что даёт:** 90 рекомендаций по написанию качественного Java-кода. Builder-паттерн,
  работа с equals/hashCode, Optional, streams -- всё с подробными объяснениями "почему".
- **Кому читать:** когда уже уверенно пишете на Java и хотите писать лучше.
  Особенно полезно перед интервью на Senior-позиции.

#### "Head First. Паттерны проектирования" -- Eric Freeman, Elisabeth Robson

- **Язык:** русский перевод доступен
- **Уровень:** начинающий / средний
- **Что даёт:** design patterns в доступном формате с картинками и аналогиями.
  Observer, Strategy, Factory, Decorator, Singleton -- паттерны, которые встречаются
  в тестовых фреймворках.
- **Кому читать:** для понимания архитектуры фреймворков и написания расширяемого
  тестового кода. Page Object -- это по сути паттерн, и понимание паттернов помогает
  проектировать свои абстракции.

**Как использовать Telegram-чаты эффективно:**
1. Перед вопросом поищите в истории чата -- скорее всего, ваш вопрос уже задавали
2. Формулируйте вопрос конкретно: что делаете, что ожидаете, что получаете, какая ошибка
3. Прикладывайте код и логи (используйте Pastebin или Gist для больших фрагментов)
4. Отмечайте тех, кто помог, благодарностью -- это мотивирует помогать снова
5. Не стесняйтесь задавать "глупые" вопросы -- все когда-то начинали

## Как эффективно использовать ресурсы

### Принцип "70-20-10"

- **70% времени** -- практика (написание кода, тестов, работа с инструментами)
- **20% времени** -- обучение через взаимодействие (чаты, code review, парное программирование)
- **10% времени** -- формальное обучение (книги, курсы, видео)

### План на неделю (пример)

| День | Активность | Ресурс |
|------|-----------|--------|
| Понедельник | 2 SQL-задачи + 1 Java-задача | HackerRank |
| Вторник | Написать 2 API-теста для публичного API | REST Assured + Reqres |
| Среда | Посмотреть 1 видео + прочитать 1 статью | YouTube + Baeldung |
| Четверг | Написать 2 UI-теста для SauceDemo | Selenide + SauceDemo |
| Пятница | Code review своего кода + рефакторинг | GitHub + собственный проект |
| Суббота | 1 глава книги + 3 задачи на LeetCode | Хорстманн + LeetCode |
| Воскресенье | Обзор Telegram-чатов + Reddit + отдых | Сообщества |

### Чек-лист: какие ресурсы освоить на каждом уровне

**Junior QA (0-6 месяцев):**
- [ ] "Тестирование dot com" -- прочитать полностью
- [ ] SQLBolt -- пройти все уроки
- [ ] Stepik "Программирование на Java" -- первые 5 модулей
- [ ] HackerRank SQL Easy -- решить 20 задач
- [ ] The Internet -- протестировать 10 страниц вручную
- [ ] Подписаться на 2-3 Telegram-чата по QA

**Middle QA Automation (6-18 месяцев):**
- [ ] Хорстманн, том 1 -- главы по ООП, коллекциям, Stream API
- [ ] Baeldung -- прочитать серии по REST Assured и JUnit 5
- [ ] Codewars -- достичь 5 kyu на Java
- [ ] HackerRank SQL Medium -- решить 15 задач
- [ ] Создать тестовый проект на GitHub с Allure и CI/CD
- [ ] Artsiom Rusau QA Life -- посмотреть серию по автоматизации

**Senior QA Automation (18+ месяцев):**
- [ ] "Effective Java" -- прочитать ключевые главы
- [ ] Документация TestContainers -- изучить и применить
- [ ] "Rapid Software Testing" -- прочитать
- [ ] Документация GitHub Actions -- освоить продвинутые возможности
- [ ] Вести блог или канал, делиться опытом
- [ ] Участвовать в open-source проектах (хотя бы issues и PR)

## Итоговые рекомендации

1. **Не пытайтесь изучить всё сразу.** Выберите 2-3 ресурса и сфокусируйтесь на них.
   Лучше глубоко изучить REST Assured, чем поверхностно -- REST Assured, Selenium,
   Playwright, Cypress и Karate одновременно.

2. **Практика важнее теории.** Прочитали главу книги -- сразу напишите код. Посмотрели
   видео -- повторите на своём проекте. Без практики знания испаряются за неделю.

3. **Создавайте портфолио на GitHub.** Каждый проект, каждое упражнение -- коммитьте
   в публичный репозиторий. Это ваше резюме, которое говорит громче любых слов.

4. **Читайте документацию.** Stack Overflow хорош для quick fix, но документация --
   для понимания. Привычка читать official docs отличает профессионала от любителя.

5. **Общайтесь с сообществом.** Задавайте вопросы, отвечайте на вопросы других,
   обсуждайте подходы. Коллективный опыт ценнее любой книги.

6. **Возвращайтесь к основам.** Даже с 3-летним опытом полезно перечитать "Тестирование
   dot com" или повторить SQL-задачи. Фундамент не бывает слишком прочным.
